Overview

Brief explanation of what we have done

Online book store in Korea is as prosperous as in any other countries in the world. It has about 36% of entire book market share in Korea, and this number is consitently growing every year. YES24 is the leading online book store in Korea, occupying about 44% of online book market. However, YES24's book recommender system is not so good that it often recommends totally irrelavant books, or even a goods that is not a book. YES24 recently opened a competition where data scientists suggest a solution for accurate book recommender, releasing the purchase history of 19,000 users within 2014. This motivated us to build a book recommeder algorithm, using the technique we've learned throughout the semester in CS109.

 

We've tried three different algorithms: one is the simplest algorithm that YES24 supposedly uses on their current website, and the others are based on k-nearest neighbor (kNN) with user similarity and book similarity. As a result, we found that kNN with user similarity has the best recommendation accuracy, which we expect will be better than the current system by 2~7%p.

Data

What we were given

The data given by YES24 contains the information of 511,742 transactions of about 19,000 users' purchase history, involving about 19,000 books. Among them, we focused on data in five categories: foreign literature, Korean literature, self-development, humanities, and religion. They are five biggest categories except study materials and magazines.

One problem of the given data is that it is the record of what people bought only; we may obtain information about which books each person likes, but it is impossible to know which books each person dislike. Usual methods in machine learning are likely to classify items into different categories, thus we needed to modify these methods to circumvent this difficulty.

Method

What we manipulated

"I will buy what you buy"

A simple algorithm which is likely to be used in current online bookstores. If another person bought what I bought, then his/her preference would be similar to mine. In other words, we recommend items that "Customers Who Bought This Item Also Bought."

Book Similarity

Some books are written by the same author or the same publisher. Then wouldn't they be similar to one another? Also sometimes their titles are similar, in some of which cases they would contain similar contents. We use this information to recommend books to customers.

User Similarity

Similar to the first method but refined further. We carefully choose how many users whose transaction record is similar to me, how importantly we consider each similar user, and how we decide a score for each book from this. Popularity of each book in general is also considered.

Result

What we have got

We applied aforementioned three different methods to the test data. Since we supposed that the first method is likely to be used currently in YES24, which we call "the baseline," we hoped to implement a better algorithm than this one. It turned out that the performance of the third method, which we call "user kNN," is remarkable. It excels the baseline method in every category by at most 7%p. Roughly it means that if we recommend 100 books to customers, then "user kNN" method would predict 7 more books they are willing to buy than the baseline.

Detailed Results

What we have really got

Here each marking denotes 2%p. The brief explanation for each method is as follows.

Baseline (the first method) for a given user, find all customers who share at least
one common book and accept the majority decision
Book kNN (the second method) use similarity between books (authors, publishers, titles, etc.)
Popularity based on popularity of each book only
User kNN (the third method) find k-customers whose patterns are the most similar to a given user
and grade a score for each book from their transactions
Weighted User kNN a mixture of "Popularity" and "User kNN"

As shown in the bar chart, the "User kNN" algorithm always surpassed other methods.
